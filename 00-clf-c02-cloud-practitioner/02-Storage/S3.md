# S3 — Simple Storage Service

## What is it?
Object storage service. Store and retrieve any amount of data (files, images, backups, logs, static websites) via the internet or AWS network.

Like a massive, infinitely scalable NAS — but accessed via HTTP/HTTPS rather than NFS/SMB. No capacity planning, no RAID, no file system to manage. You store objects in buckets.

---

## Key Concepts
- **Bucket** — container for objects. Name must be globally unique across all AWS.
- **Object** — a file + its metadata. Max object size = 5TB.
- **Key** — the object's path within a bucket (e.g., images/photo.jpg)
- **Region** — buckets are created in a specific region
- **Versioning** — keep multiple versions of an object
  (protection against accidental delete/overwrite)
- **Static Website Hosting** — serve HTML/CSS/JS directly from S3

## Storage Classes
| Class | Use Case | Retrieval |
|---|---|---|
| **S3 Standard** | Frequently accessed data | Instant |
| **S3 Standard-IA** | Infrequent access, still needs fast retrieval | Instant |
| **S3 One Zone-IA** | Infrequent, non-critical data (single AZ only) | Instant |
| **S3 Glacier Instant** | Archive, accessed a few times per year | Instant |
| **S3 Glacier Flexible** | Archive, accessed 1-2x per year | Minutes to hours |
| **S3 Glacier Deep Archive** | Long-term archive, rarely accessed | Up to 12 hours |
| **S3 Intelligent-Tiering** | Unknown or changing access patterns | Instant |

## When to Use It ✅
- Store any type of file or object at any scale
- Static website hosting
- Backup and restore
- Data lake and analytics input
- Distributing files globally (pair with CloudFront for performance)

## When NOT to Use It ❌
- Need a mountable file system → use EFS
- OS or application disk for EC2 → use EBS
- Block storage → use EBS

---

## Good to Know
- S3 is **object storage** — not block storage, not a file system
- Bucket names are **globally unique** across all AWS accounts worldwide
- S3 is **infinitely scalable** — no storage limit
- By default, all objects and buckets are **private**. Public access
  requires explicit configuration.
- S3 has **11 nines of durability** (99.999999999%) — data is
  automatically replicated across multiple AZs within a region
- Glacier storage classes are for **archival** — not for data you
  need to access quickly or frequently

---

## Easy to Mix Up
- S3 is NOT a file system — you cannot mount standard S3 like a drive
- Versioning must be enabled explicitly — it is NOT on by default
- **S3 Standard-IA** still gives instant retrieval — the difference
  is cost, not speed. You pay less per GB stored but more per retrieval.
- S3 bucket names are globally unique but buckets themselves are
  region-specific (data stays in the region you chose)
